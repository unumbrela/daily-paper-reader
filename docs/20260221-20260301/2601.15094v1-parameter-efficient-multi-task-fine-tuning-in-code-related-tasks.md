## 摘要
大语言模型（LLMs）已被证明在自动化软件工程任务方面非常有效，通过连接自然语言和代码语义，在代码生成和摘要方面取得了显著成果。然而，它们的规模带来了巨大的计算成本，使得全量微调变得不切实际。诸如 QLoRA 等参数高效微调（PEFT）方法能够以较低的资源需求实现高效的专业化。最近的研究表明，经过 QLoRA 优化的代码大模型（LCMs）在各种任务中表现强劲，但当单个模型针对多个代码相关任务进行 QLoRA 微调时，这种有效性是否依然存在尚不明确。多任务微调与 QLoRA 优化之间的相互作用，以及迁移学习如何影响生成产物的正确性和质量，在很大程度上仍未得到探索。我们研究了跨三个代表性任务（代码生成、翻译和摘要）的多任务 QLoRA 微调。我们通过基于执行和基于相似性的指标评估功能正确性，并辅以全面的代码质量分析——这是先前工作中很大程度上被忽视的一个方面。我们的研究结果表明，多任务 QLoRA 有效地利用了迁移学习，相对于单任务 QLoRA 和多任务全量微调，实现了具有竞争力或更优越的性能。较大的模型在正确性和质量之间表现出更一致的平衡，而较小的模型虽然保留了功能性，但质量相关问题的发生率更高。

## Abstract
Large Language Models (LLMs) have proven highly effective in automating software engineering tasks, bridging natural language and code semantics to achieve notable results in code generation and summarization. However, their scale incurs substantial computational costs, making full fine-tuning impractical. Parameter-Efficient Fine-Tuning (PEFT) methods like QLoRA enable efficient specialization with lower resource demands. Recent studies show QLoRA-optimized Large Code Models (LCMs) perform strongly across diverse tasks, yet it remains unclear whether this effectiveness persists when a single model is QLoRA fine-tuned for multiple code-related tasks. The interaction between Multi-task fine-tuning and QLoRA optimization, and how transfer learning affects correctness and quality of generated artifacts, remains largely unexplored. We investigate Multi-task QLoRA fine-tuning across three representative tasks: code generation, translation, and summarization. We evaluate functional correctness through execution-based and similarity-based metrics, complemented by comprehensive code quality analysis--an aspect largely overlooked in prior work. Our findings show that Multi-task QLoRA effectively leverages transfer learning, achieving competitive or superior performance relative to both Single-task QLoRA and Multi-task full fine-tuning. Larger models demonstrate more consistent balance between correctness and quality, whereas smaller models preserve functionality but exhibit a higher incidence of quality-related issues.

*   **核心问题**：
    1.  多任务 QLoRA 微调是否能通过迁移学习提升性能？
    2.  多任务微调是否会破坏模型的表示稳定性，从而影响生成代码的**功能正确性**（如 pass@1）和**非功能质量**（如复杂度、可维护性）？

### 2. 方法论
*   **核心思想**：利用 QLoRA 技术在单个模型中同时注入针对多种代码任务（生成、翻译、摘要）的低秩适配器（Adapters），实现资源节约型的高性能多任务学习。
*   **关键技术细节**：
    *   **基础模型**：选用 Qwen2.5-Coder-Instruct 系列（0.5B、1.5B、3B 三个尺寸）。
    *   **QLoRA 配置**：秩 $r=8$，缩放因子 $\alpha=16$，丢弃率 0.1。适配器注入到 Transformer 的所有线性层（q, k, v, o, gate, up, down_proj）。
    *   **量化技术**：使用 4-bit NormalFloat (NF4) 数据类型、双量化（Double Quantization）和分页优化器（Paged Optimizers）。
    *   **训练策略**：对比了单任务 QLoRA (ST-QLoRA)、多任务 QLoRA (MT-QLoRA) 和多任务全量微调 (MT-FFT)。

### 3. 实验设计
*   **任务与数据集**：
    *   **代码生成 (NL-to-Code)**：使用 CodeXGLUE 训练，CoderEval (Python/Java) 评估。
    *   **代码翻译 (Code-to-Code)**：Java 与 C# 互译，使用 CodeXGLUE 数据集。
    *   **代码摘要 (Code-to-NL)**：CodeXGLUE 训练，CoderEval 评估。
*   **Benchmark 指标**：
    *   **正确性**：pass@1（执行评估）、CodeBLEU、BLEU、METEOR、ROUGE-L、BERTScore、SIDE。
    *   **代码质量（静态分析）**：使用 Lizard（复杂度）、Pylint/PMD（风格/错误）、SonarCloud（可维护性）、Roslyn（C# 语法）。
    *   **摘要质量**：引入 **LLM-as-judge**（使用 GPT-5 Mini）评估内容充分性、简洁性和流畅度。
*   **对比方法**：ST-QLoRA vs MT-QLoRA 以及 MT-QLoRA vs MT-FFT。

### 4. 资源与算力
*   **硬件环境**：使用了 **8 张 NVIDIA A40 GPU**（每张 46GB 显存）。
*   **软件环境**：Ubuntu 22.04, CUDA 12.2。
*   **训练参数**：训练 5 个 Epoch，有效 Batch Size 为 32，学习率 $1 \times 10^{-4}$。QLoRA 使用 8-bit AdamW 优化器，FFT 使用标准 AdamW 并开启梯度检查点（Gradient Checkpointing）。

### 5. 实验数量与充分性
*   **实验规模**：共训练并评估了 **15 个模型**（3 种尺寸 $\times$ 5 种配置）。
*   **充分性**：实验覆盖了从 0.5B 到 3B 的模型梯度，涵盖了三种主流编程语言（Python, Java, C#），并从功能正确性和非功能质量两个维度进行了双重验证。
*   **客观性**：使用了非参数统计检验（McNemar 检验和 Wilcoxon 符号秩检验）以及 Holm-Bonferroni 校正，确保观察到的性能差异具有统计学意义。

### 6. 主要结论与发现
*   **多任务优势**：MT-QLoRA 在多项任务中表现出与 ST-QLoRA 相当甚至更优的性能，证明了任务间的正向迁移。
*   **规模效应**：**模型越大，平衡能力越强**。3B 模型在 MT-QLoRA 下能同时提升正确性和代码质量（如 Python 复杂度降低 32%）；而 0.5B 小模型在多任务下容易出现质量下降。
*   **对比 FFT**：MT-QLoRA 在代码生成和 Python 摘要任务上甚至优于 MT-FFT，且显存占用大幅降低。
*   **语言差异**：Python 任务从多任务微调中获益最明显；Java 摘要任务在单任务微调下表现略好。
*   **质量发现**：参数高效微调并不一定会牺牲代码质量，在足够大的模型规模下，它能生成更简洁、复杂度更低的代码。

### 7. 优点与亮点
*   **多维度评估**：不仅关注 pass@1 等硬指标，还首次在大规模 PEFT 研究中引入了极其详尽的静态分析（5 种工具）和 LLM 主观评估。
*   **实用性强**：验证了在资源受限情况下，使用一个统一的 QLoRA 模型处理多种软件工程任务的可行性。
*   **开源贡献**：提供了完整的复制包，包括数据集、微调模型和评估脚本。
