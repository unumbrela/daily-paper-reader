## 摘要
检索增强生成（RAG）已成为通过引入外部知识来缓解大语言模型（LLM）幻觉问题的主流范式。然而，如何有效地整合和解读散布在噪声文档中的关键证据，仍然是现有 RAG 系统面临的一项重大挑战。在本文中，我们提出了 GraphAnchor，这是一种新颖的图锚定知识索引方法，它将图结构从静态的知识表示重新定义为主动的、演进的知识索引。GraphAnchor 在迭代检索过程中增量更新图，以锚定显著的实体和关系，从而产生一个结构化索引，指导 LLM 评估知识的充分性并构建后续的子查询。最终答案是通过共同利用所有检索到的文档和最终演进的图生成的。在四个多跳问答基准测试上的实验证明了 GraphAnchor 的有效性，并揭示了 GraphAnchor 能够调节 LLM 的注意力，从而更有效地关联分布在检索文档中的关键信息。所有代码和数据均可在 https://github.com/NEUIR/GraphAnchor 获取。

## Abstract
Retrieval-Augmented Generation (RAG) has emerged as a dominant paradigm for mitigating hallucinations in Large Language Models (LLMs) by incorporating external knowledge. Nevertheless, effectively integrating and interpreting key evidence scattered across noisy documents remains a critical challenge for existing RAG systems. In this paper, we propose GraphAnchor, a novel Graph-Anchored Knowledge Indexing approach that reconceptualizes graph structures from static knowledge representations into active, evolving knowledge indices. GraphAnchor incrementally updates a graph during iterative retrieval to anchor salient entities and relations, yielding a structured index that guides the LLM in evaluating knowledge sufficiency and formulating subsequent subqueries. The final answer is generated by jointly leveraging all retrieved documents and the final evolved graph. Experiments on four multi-hop question answering benchmarks demonstrate the effectiveness of GraphAnchor, and reveal that GraphAnchor modulates the LLM's attention to more effectively associate key information distributed in retrieved documents. All code and data are available at https://github.com/NEUIR/GraphAnchor.

### 2. 方法论：核心思想与技术细节
GraphAnchor 的核心是将图作为主动的索引工具，其流程分为以下几个关键步骤：
*   **迭代检索与图演化**：
    1.  **检索（Retrieval）**：根据当前查询 $q_{t-1}$ 检索相关文档 $D_t$。
    2.  **图更新（Graph Update）**：LLM 根据新检索的文档、先前的推理轨迹和旧图 $G_{t-1}$，增量式地提取实体和 RDF 三元组（关系），生成更新后的图 $G_t$。
    3.  **知识评估（Sufficiency Judgement）**：LLM 基于当前的图索引和文档，判断现有信息是否足以回答原始问题。
    4.  **子查询生成（Sub-query Generation）**：若信息不足，LLM 利用图结构识别知识缺口，生成下一个子查询 $q_t$，进入下一轮迭代。
*   **最终答案生成**：当判断知识充分或达到最大迭代次数（如 4 次）时，LLM 结合所有检索到的原始文档和最终演化出的结构化图索引，生成最终答案。
*   **图的线性化表示**：为了让 LLM 处理图，GraphAnchor 将实体及其属性、RDF 三元组（头实体-关系-尾实体）转化为文本序列，并用特殊的 `<graph>` 标签包裹。

*   **检索质量提升**：图索引能帮助模型生成更多样化、重复率更低的子查询，从而在迭代中获取更互补的知识。

### 7. 优点与亮点
*   **角色转变**：将图从单纯的“知识展示”转变为“动态索引”，这一视角非常新颖，解决了 RAG 中证据分散的问题。
*   **可解释性强**：通过图的演化过程，可以清晰地观察到模型是如何一步步补全逻辑链条的。
*   **即插即用**：该方法不需要对 LLM 进行微调，通过精心设计的 Prompt 即可在现有的强大 LLM 上实现。

### 8. 不足与局限
*   **推理开销**：迭代检索和每一步的图构建带来了较高的 Token 消耗和时间延迟，不适合对实时性要求极高的场景。
*   **依赖模型能力**：图的质量高度依赖于底层 LLM 的信息提取能力。如果模型在第一步提取了错误的实体，可能会导致后续检索偏离方向。
*   **表示形式单一**：目前采用线性化文本表示图，对于极其复杂的超大规模图结构，可能会受限于 LLM 的上下文窗口长度。
